## 작업 종료 시 규칙

세션 끝나기 전, Notion 작업 로그 데이터베이스에 오늘 작업 내용을 새 페이지로 추가해줘.

- database_id: 3e22434b4be049a8b5d4c0410216b80e
- 작업 시간과 컨디션(집중됨/보통/힘들었음)은 사용자에게 물어봐서 입력

### 속성(properties) 입력
- 제목: 오늘 날짜 + 작업 요약 한 줄 (예: 2026-06-07 — 종목 추천 고도화)
- 날짜: 오늘 날짜
- 오늘 한 일 / 된 것 / 안 된 것 / 다음에 할 것 / 프로젝트 / 작업 시간 / 컨디션

### 본문(content) 작성 규칙 — 가독성 중요
속성만 채우지 말고, 반드시 본문도 아래 형식으로 상세하게 작성할 것.

1. 맨 위에 "## 📋 작업 상세 내용" 헤딩
2. 작업을 항목별로 나누고 각각 "### 번호. 제목 ✅" 형식의 소제목
3. 소제목 사이에 구분선(---) 넣기
4. 각 항목은 불릿으로 구체적으로 (무엇을, 왜, 어떻게)
5. 비율·종목·수치 등 비교 정보는 표(table)로 정리
6. 트러블슈팅이 있으면 "### 🔧 트러블슈팅" 섹션
7. 새로 배운 개념이 있으면 "### 📚 배운 개념" 섹션
8. 코드/명령어는 인라인 코드(백틱)로 표시

### 톤
- 간결하지만 나중에 다시 봐도 이해되도록 맥락 포함
- 단순 나열이 아니라 "왜 그렇게 했는지" 이유를 남길 것
